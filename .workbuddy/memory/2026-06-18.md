## 修复：找钓点页已审核公开钓点仍显示"私人"标签

**根因**：两处 bug 叠加
1. `my-spot.service.ts getMyMapSpots`：本地缓存（`localMySpots`）优先级高于远程数据，
   导致审核通过后 reviewStatus 仍读取本地旧缓存（PENDING）而非服务端真实值（APPROVED）。
2. `spots/index.ts mergeSpots`：`mySpots` 后写覆盖 `publicSpots`，
   已公开审核通过的钓点被 mySpots 版本（isMine=true, 可能含旧状态）覆盖。

**修复**：
- `getMyMapSpots`：remoteList 优先，localDetails 仅补充 remoteIds 中不存在的条目。
- `mergeSpots`：mySpots 先写，publicSpots 后写覆盖（保留 isMine 标志），
  确保进入公开列表的钓点以服务端权威数据为准。

## 修复：我的发布页面时间始终显示"刚刚"

**根因**：模板 `{{post.createdAtText || '刚刚'}}` 依赖 `createdAtText` 字段，
但 `toPost()` 从未计算该字段，后端返回的 `createdAt`（ISO 8601）只存成了原始字符串。
即使数据正确，模板也始终走 fallback 显示"刚刚"。

**修复**：
- `models/index.ts`：`CommunityPost` 接口补充 `createdAtText?: string`
- `services/post.service.ts`：新增 `formatTime(iso)` 工具函数，支持三个粒度
  - 今天 → `今天 HH:mm`
  - 昨天 → `昨天 HH:mm`
  - 更早 → `2026年6月15日 HH:mm`
- `toPost()` 和三个 mock fallback 路径（getPosts / getUserPosts / getPostById）均补上 `createdAtText` 计算

涉及文件：models/index.ts、services/post.service.ts

### 修复循环依赖（@Lazy → 重构 PostConverter）
- 初版用 `@Lazy` 打破 PostService ⇄ BrowseHistoryService 循环依赖
- 用户要求不使用 @Lazy，改用重构方案
- 抽取 `PostConverter`（@Component）：只依赖 Mapper（PostImageMapper/UserMapper/FishingSpotMapper/UserFollowMapper/PostLikeMapper），无业务依赖
- `PostService`：去掉私有 `toVO()`、去掉 @Lazy，改为注入 `PostConverter` 调 `postConverter::toVO`
- `BrowseHistoryService`：去掉对 `PostService` 的依赖，改注入 `PostConverter` + 已有 `PostMapper`
- 依赖关系：PostService → BrowseHistoryService → PostConverter；PostService → PostConverter（无循环）
- 最佳实践：service 间共享的 Entity→VO 转换应抽到独立 Converter 组件，避免循环依赖
